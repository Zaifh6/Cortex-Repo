Lead Generation · n8n + monday.com + Groq
Lead Capture Automation Pipeline

End-to-end pipeline capturing form submissions, creating CRM entries, writing AI-personalized emails via Groq, and updating lead status — all within seconds.

▶ Video Demo ↗ Live Case Study

The Problem

Every form submission required manual CRM entry, a hand-written follow-up email, and a status update. Leads were going cold before anyone responded.

What We Built
  1. Google Form triggers n8n via Google Sheets on each new submission.
  2. Code node formats data; HTTP Request creates a monday.com item via GraphQL API with status "New Lead."
  3. Groq (Llama 3.3 70B) writes a personalized outreach email based on name, company, and stated need.
  4. Gmail sends the AI-written email. A final API call sets CRM status to "Contacted."
Outcome

Zero manual touchpoints. Personalized email delivered in under 5 seconds. CRM always accurate with no human input.

Cloud Security · n8n + AWS IAM + Slack
AWS IAM Dormant User Alert System

Weekly automated audit of all AWS IAM users — flags accounts inactive 90+ days and posts structured Slack alerts to security teams, eliminating manual cloud security reviews.

Internal Deployment
0 hrs
Manual Audit Time
100%
IAM Coverage
90d

The Problem

Inactive AWS IAM credentials are a silent attack surface. Manual audits are tedious, inconsistent, and create SOC 2, ISO 27001, and CIS compliance gaps.

What We Built
  1. Weekly n8n scheduler fires automatically — also manually triggerable on demand.
  2. AWS ListUsers API retrieves every IAM user in the account.
  3. GetUser fetches PasswordLastUsed timestamp for each account.
  4. Filter node removes service-linked accounts and noisy entries.
  5. Accounts inactive 90+ days trigger a Slack alert with ARN, username, last activity, and remediation steps.
Outcome

Precise weekly Slack report of every dormant credential with zero effort. Compliance gaps surfaced before audits. DevOps IAM review time dropped to zero.

The Problem

Generic product descriptions hurt SEO and conversions. Writing unique copy for hundreds of products is expensive — and every new product brings the bottleneck back.

What We Built
  1. Make.com monitors Shopify and triggers on every product creation or update.
  2. Product title, category, and tags are passed to GPT-4 with a custom e-commerce conversion prompt.
  3. GPT-4 Markdown output is converted to properly formatted HTML via a parser.

Autonomous LinkedIn Content System

Generates, approves, and publishes AI-written LinkedIn posts Mon–Thu with Telegram human-in-the-loop approval — full automation without losing editorial control.

The Problem

LinkedIn is high-ROI for B2B professionals but daily writing is the first thing dropped when life gets busy. The client needed consistent weekday presence without losing control over what gets published.

What We Built
  1. n8n scheduler fires at 8AM daily; logic node exits on weekends.
  2. A random prompt is pulled from a curated bank — no two posts start from the same angle.
  3. GPT generates a complete post tuned to the client's voice and audience.
  4. Telegram bot sends the draft; client approves or rejects with one tap. Nothing publishes without approval.
  5. After approval, workflow waits for a randomized optimal window, publishes via LinkedIn API, confirms via Telegram.
Outcome

4 posts per week consistently, zero writing effort, full editorial control. Review takes under 10 seconds per post.

Infrastructure · n8n + GoDaddy + Cloudflare
New Project Bootstrap Blueprint

A 28-node, 4 sub-workflow n8n automation that registers a domain, configures Cloudflare DNS and SSL, provisions Resend outbound email, and sets up Mailgun inbound routing — from one trigger.

↗ Live Case Study
28
Nodes · 4 Sub-flows
<5m
Full Stack Live
1
Trigger to Done
The Problem

Spinning up a project — domain, DNS, SSL, outbound and inbound email — takes hours across 4+ platforms. Every new client launch faced the same repetitive bottleneck.

What We Built
  1. GoDaddy: Check availability → register → update nameservers to Cloudflare.
  2. Cloudflare: Add zone → A record → CNAME → Universal SSL → SPF/DMARC records.
  3. Resend: Add domain → extract DKIM → push TXT to Cloudflare → verify → send test email.
  4. Mailgun: Add domain → catch-all routing → forwarding rules → autoresponder → MX records.
  5. Summary report merges all outputs into a pass/fail checklist emailed to the owner.
Outcome

2–3 hours of manual setup now completes in under 5 minutes from a single trigger. Fully reusable for every new project or client onboarding.
